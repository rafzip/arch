## What this repo is

Arch Linux dotfiles for a Hyprland desktop. `install-dots.sh` symlinks everything under `.config/` into `$HOME/.config/`. The repo is also served via Vercel as raw text (see `vercel.json`).

## IMPORTANT INFO

check the docs every time before doin config stuff https://wiki.hypr.land/  https://archlinux.org/ 

check hypr configs for errors with `hyprctl configerrors`

To reload a running Hyprland session without logging out:
```bash
~/.config/hypr/scripts/reload.sh   # hyprctl reload + restart waybar + hypridle
```

Or from inside Hyprland: `Super+R`.

## Config structure

`hyprland.conf` is the entry point — it sources these files in order:

| File | Purpose |
|---|---|
| `env.conf` | Wayland/Qt env vars, cursor theme, NVIDIA hybrid flags |
| `autostart.conf` | `exec-once` daemons (waybar, swaync, awww, pypr, hypridle…) |
| `monitors.conf` | Monitor layout (generated by `hyprmon`; also mirrored in `monitors.json`) |
| `keybinds.conf` | All `bind`/`bindel`/`bindm` entries |
| `looks.conf` | Gaps, borders, blur, animations |

`hypridle.conf` and `hyprlock.conf` are standalone (not sourced from hyprland.conf).

## Hardware notes

- **Backlight**: AMD GPU, device `amdgpu_bl2` — hardcoded in `brightness-osd.sh` and `hypridle.conf`. Change both if the device name differs.
- **Hybrid GPU**: NVIDIA dGPU offload via `toggle-gpu-offload.sh`. `AQ_FORCE_LINEAR_BLIT` is set in `env.conf`; uncomment `AQ_DRM_DEVICES` if GPU priority needs forcing.
- **Monitors**: `monitors.conf` is authoritative for Hyprland; `monitors.json` is the source for the `hyprmon` tool's workspace memory.

## Waybar

Config at `.config/waybar/config.jsonc`. Custom modules poll scripts in `.config/waybar/scripts/`:
- `gpu.sh` — NVIDIA utilization via `nvidia-smi` (0.5s timeout, graceful fallback)
- `fans.sh` — reads first `fan*_input` from hwmon
- `net.sh` — per-second throughput on the default route interface

Focus mode (`Super+M`) toggles waybar visibility via `SIGUSR1` and zeroes gaps/rounding live with `hyprctl --batch`.
